## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.02-01 | Tampering | createOrder — client-supplied walletAmount | mitigate | Server-side Zod schema enforces `min(0)`; server re-queries wallet balance before deducting. Client cannot set arbitrary amounts. |
| T-02.02-02 | Elevation of Privilege | createOrder — TOCTOU wallet balance race | mitigate | Atomic `findOneAndUpdate` with `balance: { $gte: walletAmount }` filter (D-11). Returns null if balance insufficient at deduct time → 409 returned, stock rolled back. |
| T-02.02-03 | Tampering | cancelOrder — unauthorized caller restoring stock | accept | cancelOrder already validates `Order.findOne({ _id, userId: req.user!.userId })` — only order owner can cancel. |
| T-02.02-04 | Denial of Service | cancelOrder — partial stock restore failure | accept | Per D-14: per-item failures are logged but do not block response. Partial failure is preferable to blocking the cancel. Monitoring via console.error. |
