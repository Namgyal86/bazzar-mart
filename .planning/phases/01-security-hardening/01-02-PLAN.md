<objective>
Harden delivery-service and notification-service: replace wildcard CORS with origin-restricted config mirroring the monolith's pattern, and add JWT verification middleware to all delivery agent routes.

Purpose: Wildcard CORS with credentials exposes these services to CSRF from any domain. Unprotected delivery routes allow unauthenticated clients to impersonate drivers, mark orders as delivered, and trigger Kafka events that update order status.

Output: Both services restricted to known origins. All delivery agent/admin routes (except /health and /track) protected by JWT.
</objective>

1. Create an `authenticateJWT` middleware function in the same file (before the route definitions). The `jsonwebtoken` import and `SECRET` constant already exist (line 8 and line 17):

```typescript
function authenticateJWT(req: express.Request, res: express.Response, next: express.NextFunction): void {
  const header = req.headers.authorization;
  if (!header?.startsWith('Bearer ')) {
    res.status(401).json({ success: false, error: 'No token' });
    return;
  }
  try {
    (req as any).user = jwt.verify(header.slice(7), SECRET);
    next();
  } catch {
    res.status(401).json({ success: false, error: 'Invalid token' });
  }
}
```
